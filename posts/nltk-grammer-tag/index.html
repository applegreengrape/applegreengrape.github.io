Text Mining | NLTK | Python

2019-10-08
#Python  #nltk  #text mining 

In my previous posts, we have crawled some ariticles and news through RSS feed and HTML tags. We played with google’s nlp api and then as a developer without too much pocket money to play with. You end up looking for cheaper and better alternatives 👉nltk (natual language toolkit).

Let’s break the sentence down and let your code to define how will it read 🤓

How about let the code understand your emotions and 👉 you to the grammer mistake you made.

Take this sentence as an example:

My english is bad and I will got the grammer wrong but I am still happy !

Let’s import all the python modules needed.

from nltk.tokenize import sent_tokenize
from nltk.tokenize import word_tokenize
from nltk import pos_tag
from nltk import RegexpParser
from nltk import RegexpTagger

Make a function to tag with regular expressions:

def rule_based_tag(sent):
    my_tags = RegexpTagger([
        (r'bad|wrong|sad', '😞'),
        (r'happy|good|nice', '😃'),
            ])

    reg_out = my_tags.tag(word_tokenize(sent))
    tag_out = pos_tag(word_tokenize(sent))
    return [(token, rtag) if rtag == '😞' or rtag == '😃' else (token, ttag) for (token, rtag), (token, ttag) in zip(reg_out, tag_out)]

tag = rule_based_tag(sent)

Now let’s add some grammer here.

grammar = r"""
  bag_grammer : {<MD>+<VBG>}
  my_emotions : {<😞>|<😃>}
  """

cp  = RegexpParser(grammar)
tree = cp.parse(tag)

tree.draw()

And here you go. They are just layers and layers of regular expressions. How cool is that 😃😃😃!

Oops…This tree.draw() is not supported emojis 😞😞😞.
